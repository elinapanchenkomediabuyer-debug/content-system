# ПЛАН СОЗДАНИЯ КОНТЕНТ-СИСТЕМЫ ДЛЯ СОЦИАЛЬНЫХ СЕТЕЙ КЛИНИКИ

## Цель проекта

Создать **готовую контент-систему** — переиспользуемый механизм, который позволяет быстро и качественно производить любые маркетинговые материалы клиники по каждой услуге.

**Важно:** цель проекта — сама система, а не её продукты.

В этом репозитории хранится **система**. Конкретные материалы (сценарии, флаеры, баннеры, посты, ТЗ дизайнеру) — это **продукты**, которые система производит по запросу. Они не являются частью системы и здесь не хранятся.

### Система ≠ продукты

| Система (хранится в репозитории) | Продукты (создаются системой по запросу) |
|---|---|
| База знаний по услугам | Сценарии роликов |
| Методика Яны Ащиной | Флаеры |
| Правила копирайтинга | Баннеры, материалы для экранов |
| Идеальные шаблоны | Посты |
| Портреты целевой аудитории | Ответы администраторам |
| Система поиска тем | Тексты для рекламы |
| Обученные проекты по пулам услуг | Планы съёмок, ТЗ дизайнеру |

Дальше описаны этапы построения **системы**.

---

## ✅ ЭТАП 1. Определение охвата системы. (готово!) (20 минут)

Определить, какие услуги покрывает система.

1. Составить полный список услуг и аппаратов.
2. Сгруппировать их в **пулы услуг**.
3. Расставить приоритет пулов согласно:
   - плану продаж;
   - действующим акциям;
   - стратегическим задачам клиники.
4. Разделить пулы по двум направлениям — **косметология** и **лечение** (по этим же направлениям строится методика и ведутся аккаунты).

Именно пулы услуг становятся основой всей системы.

> **Примечание.** Продуктовое планирование (сколько роликов выпускать, как распределить их между аккаунтами — например, 30 косметология / 15 лечение) относится к производству продуктов, а не к построению системы. Оно выполняется уже при работе с готовой системой.

**Результат этапа: список услуг с разделением на косметологию и лечение.**

---

2. Лечение.

В каждом собрать максимально подробно:

- позиционирование;
- стиль коммуникации;
- структура роликов;
- способы удержания внимания;
- работа с болями;
- экспертность;
- продажи;
- призывы к действию;
- особенности медицинского копирайтинга;
- рекомендации по построению сценариев.

Эти документы становятся главными методическими материалами системы.

## ЭТАП 5. Создание базы знаний. 

Для каждого пула услуг выполнить полный цикл.

### 5.1. Собрать сырой материал.

Собрать максимально возможное количество информации:

- страницы сайта;
- описания услуг;
- описания аппаратов;
- инструкции;
- методические материалы;
- акции;
- FAQ;
- отзывы;
- кейсы;
- фотографии;
- посты Telegram;
- статьи;
- внутренние документы;
- любые дополнительные материалы.

**Важно:** информация собирается сразу по целой услуге, а не по отдельным темам.

### 5.2. Оформить по шаблону документа по услуге.

Каждую услугу оформить строго по утверждённому шаблону (пункт 4.1).

### 5.3. Автоматизировать оформление.

Пока собирается информация по следующей услуге, Claude оформляет предыдущую. Получается конвейер.

В результате появляется единая база знаний по каждой услуге.

## ЭТАП 7. Система поиска тем

Заложить в систему методику поиска тем — как переиспользуемый инструмент, а не готовый список.

Последовательность:

```
Услуга
   ↓
Портрет клиента
   ↓
Боли
   ↓
Возражения
   ↓
Желания
   ↓
Интересы
   ↓
Темы
```

В систему входит:

- методика (эта последовательность);
- шаблон поиска тем (пункт 4.4).

> Конкретные списки тем и их распределение между аккаунтами формируются уже при производстве продуктов — на основе этой методики.

## ЭТАП 8. Сборка проектов Claude по пулам услуг

Собрать саму рабочую систему.

**Единый главный репозиторий проекта** (этот репозиторий) содержит общую часть:

- методику Яны Ащиной;
- правила медицинского копирайтинга;
- принципы Максима Ильяхова;
- общие инструкции;
- универсальные промпты;
- шаблоны документов.

**Отдельный проект Claude для каждого пула услуг.** Например:

- SMAS
- Лазерная эпиляция
- RF-лифтинг
- Инъекционная косметология
- Лечение акне
- Трихология
- и т.д.

Внутри каждого проекта разместить:

- документ по услуге;
- портреты ЦА;
- шаблон поиска тем;
- методику Яны Ащиной;
- шаблон сценария;
- правила копирайтинга;
- примеры удачных сценариев (эталоны стиля).

В результате каждый проект становится экспертом только по своей услуге и готов производить любые материалы по ней.

## ЧТО ПРОИЗВОДИТ ГОТОВАЯ СИСТЕМА

Это **продукты**, а не часть системы. Они создаются по запросу и в репозитории не хранятся. С помощью готовой системы можно быстро и без повторной подготовки создавать по любой услуге:

- сценарии роликов;
- посты;
- флаеры;
- баннеры;
- материалы для экранов;
- ответы администраторам;
- тексты для рекламы;
- технические задания дизайнеру;
- планы съёмок;
- любые другие маркетинговые материалы.

Главная ценность — **масштабируемость**: система один раз собирается и дальше позволяет производить любой объём материалов быстро, качественно и в едином стиле.
